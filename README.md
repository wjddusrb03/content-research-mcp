An MCP server that combines **Naver DataLab trends + Blog/News search + Papago translation + Unsplash images** into a single research pipeline for content creators. This is NOT just another API wrapper -- it's a full research assistant that turns one question into a structured report.

## Why This Exists

Writing a blog post? Creating a YouTube script? You already know the drill:

1. Check search trends to validate the topic
2. Read top blog posts to find popular angles
3. Scan recent news to stay current
4. Find royalty-free images to use
5. Translate sources if needed

That's 4 different tabs, 4 different APIs, 30+ minutes of busywork. **content-research-mcp** does all of it in a single tool call.

## Features

| Tool | What It Does |
|------|-------------|
| `research_topic` | **The killer feature.** All-in-one research: trends + blogs + news + images combined into a single structured report |
| `trend_keywords` | Compare search trends for up to 5 keywords via Naver DataLab |
| `search_blogs` | Search Naver blogs for popular content and angles |
| `search_news` | Get the latest news articles from Naver |
| `translate_text` | Translate text with Naver Papago (en/ko/ja/zh-CN and more) |
| `search_images` | Search free-to-use images on Unsplash |
| `api_status` | Check which APIs are configured — missing keys are skipped gracefully |

## Installation

### Quick Setup (Recommended)

```bash
git clone https://github.com/wjddusrb03/content-research-mcp.git
cd content-research-mcp
pip install -r requirements.txt
python setup_wizard.py
```

The setup wizard will:
1. Ask for your API keys (interactive prompts)
2. Create a `.env` file automatically
3. Register the MCP server in Claude Desktop config
4. Back up your existing config before making changes

Restart Claude Desktop after setup, and you're good to go.

### Manual Setup

If you prefer to configure things yourself:

**1. Install dependencies**

```bash
git clone https://github.com/wjddusrb03/content-research-mcp.git
cd content-research-mcp
pip install -r requirements.txt
```

**2. Create your `.env` file**

```bash
cp .env.example .env
```

Edit `.env` with your keys:

```
NAVER_CLIENT_ID=your_client_id
NAVER_CLIENT_SECRET=your_client_secret
UNSPLASH_ACCESS_KEY=your_access_key
```

**3. Add to Claude Desktop config**

Open your Claude Desktop configuration file:

- **Windows:** `%APPDATA%\Claude\claude_desktop_config.json`
- **macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`
- **Linux:** `~/.config/claude/claude_desktop_config.json`

Add the server entry under `mcpServers`:

```json
{
  "mcpServers": {
    "content-research": {
      "command": "python",
      "args": ["/full/path/to/content-research-mcp/server.py", "--env", "/full/path/to/content-research-mcp/.env"]
    }
  }
}
```

Restart Claude Desktop.

## API Key Setup

All APIs used are **free tier**. No credit card required.

### 1. Naver Developer (for trends, blogs, news, translation)

1. Go to [developers.naver.com](https://developers.naver.com)
2. Sign in with your Naver account
3. Go to **Application** > **Register Application**
4. Application name: anything you like (e.g., "content-research")
5. Select APIs: **Search**, **DataLab (Search Trend)**, **Papago Translation**
6. Set the service URL to `http://localhost`
7. Copy **Client ID** and **Client Secret**

### 2. Unsplash (for free images)

1. Go to [unsplash.com/developers](https://unsplash.com/developers)
2. Click **New Application**
3. Accept the terms and create the app
4. Copy the **Access Key**

## Tool Reference

### `research_topic`

The main tool. Generates a full research report by calling all 4 APIs.

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `topic` | string | *(required)* | Research topic in Korean |
| `english_topic` | string | `""` | Same topic in English (for image search). Falls back to `topic` if empty. |

### `trend_keywords`

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `keywords` | list[str] | *(required)* | Keywords to compare (max 5) |
| `months` | int | `6` | How many months of data to analyze |

### `search_blogs`

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `query` | string | *(required)* | Search keyword |
| `count` | int | `5` | Number of results (max 20) |

### `search_news`

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `query` | string | *(required)* | Search keyword |
| `count` | int | `5` | Number of results (max 20) |

### `translate_text`

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `text` | string | *(required)* | Text to translate |
| `source` | string | `"en"` | Source language code |
| `target` | string | `"ko"` | Target language code |

Supported languages: `en`, `ko`, `ja`, `zh-CN`, and more.

### `search_images`

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `query` | string | *(required)* | Search keyword (English recommended) |
| `count` | int | `5` | Number of images (max 10) |
